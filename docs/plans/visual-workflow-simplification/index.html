Hermes Workflows: shrink the framework surface, move the ugly stuff down.

A visual map of what is wrong, what the framework should own, and the safest PR sequence. This is the thing the Markdown plan needed to be: a reviewable artifact, not a wall of prose.

Verdict
Skylar’s critique is right: the framework abstracted too little of the painful stuff and exposed too much ceremony.
Call: stop expanding the giant content/code/event workflows. First make the authoring surface boring: typed inputs, review requests, artifacts, prompt files, workspace-aware agents.

Kill list

Concepts to remove from the authoring surface

kill`PUBLIC_APPROVAL_GATES`
Derive review/gate state from runtime events. No upfront topology theater.
kill`approver`, `human:*`, `agent:*`
Core should model review requests and provenance, not people/agent identity policy.
killworkflow-local `from_value`
The framework parses raw JSON into typed workflow input before execution.
killmagic render dicts
Use base Artifact + derived renderable types with required metadata.
PR sequence

Small-start migration, not a big-bang rewrite

1Types
`JsonValue`, `JsonObject`, shared normalizer, field descriptions.
2Input parsing
Engine parses raw JSON into dataclass/typed inputs before workflow body.
3Review model
Neutral `ReviewRequest` / `ReviewResponse`; legacy approval as adapter alias.
4Artifacts
Base artifact metadata + Markdown/JSON/File/Link/Python derived types.
5Prompt files
`prompt_file()` with template/rendered hashes; move giant f-strings out.
6Workspace agents
`agent(..., workspace_dir=...)`; optional worktree helper later.
7Prune examples
Tiny starter examples; big lanes move to advanced/private/presentation.
What framework owns
Input boundary
raw → typed, with validation errors before decider execution.
Review boundary
actions, schemas, policy, provenance; no approver identities.
Artifact boundary
metadata, render descriptor, hash/reference, legacy normalization.
Execution boundary
agent workspace dir, runner cwd, prompt template receipts.
What workflow owns
Business sequence
what happens first, second, next; not how UI should render it.
Domain decisions
what needs review, what counts as done, what outputs matter.
Small structured state
typed inputs/outputs, not giant Markdown packets assembled inline.
What UI/plugin owns
Rendering
display declared artifact descriptors; stop guessing from random keys.
Adapter policy
auth/session/user provenance stays outside core workflow semantics.
Progressive disclosure
show active review/tasks first; hide implementation metadata until needed.
